# Security Policy

## Overview

MITM Railway Attack Simulator is an educational cybersecurity simulation demonstrating how Man-in-the-Middle (MITM) attacks can impact railway control systems.

This project is intended for **learning and demonstration purposes only** and does not represent a real-world railway system.

---

## Supported Versions

As this is a non-production educational project, there are no officially maintained versions.

The latest version available in the repository should be considered the most up-to-date.

---

## Reporting a Vulnerability

If you discover a potential security issue in this project, please report it responsibly.

You can:

- Open an issue on GitHub  
- Or contact the repository owner directly  

Please include:
- A description of the issue  
- Steps to reproduce  
- Potential impact  

---

## Security Considerations

This project simulates cyberattacks but does **not** implement real attack techniques or exploits.

Key limitations:

- No real network communication is used  
- No actual system vulnerabilities are exposed  
- All attacks are simulated for educational purposes  

---

## Responsible Use

This project is intended for:

- Education  
- Demonstrations  
- Understanding cybersecurity concepts  

It should **not** be used for malicious purposes.

---

## Built-in Security Features

The simulation includes basic security concepts such as:

- Data validation  
- Attack detection  
- Command integrity checks  

These are simplified representations of real-world security mechanisms.
